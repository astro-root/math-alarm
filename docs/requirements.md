## 1. アラーム機能

### 1.1 基本仕様
- 指定時刻に鳴動。数学の問題（設定数）に正解するまで停止不可。
- ユーザーが問題数・出題範囲・難易度・制限時間を設定可能。
- ★**スヌーズ機能は提供しない**（コンセプトと矛盾するため明確に非搭載と定義）。
- ★複数アラームの設定に対応（平日/休日パターン等）。
- ★端末再起動後もアラームが有効であること（Android: `BOOT_COMPLETED`で再登録／iOS: ローカル通知の再スケジュール）。

### 1.2 ★信頼性に関する現実的な制約（重要・要合意）
以下は技術的制約として要件に明記し、ユーザーにも正直に伝える方針とする。

| 項目 | Android | iOS |
|---|---|---|
| 強制終了(kill)後の復帰 | フォアグラウンドサービス+`SCHEDULE_EXACT_ALARM`である程度可能。ただし一部メーカー端末(MIUI/EMUI等)は独自制御で不確実 | **原理的に不可能**。ユーザーへの説明・警告表示が必須 |
| 音量0/マナーモード突破 | 可能（アラーム用オーディオストリームを使用） | `Critical Alerts` entitlement必須（審査は医療・安全用途中心で通過は不確実）。**通過しない前提で設計** |
| DND(集中モード)突破 | 可能 | 同上。Critical Alerts前提 |
| フルスクリーンでの強制表示 | `USE_FULL_SCREEN_INTENT`（Android14+は用途申告が必要） | Time Sensitive通知＋ロック画面上のインタラクティブ通知で代替 |

→ 対応方針：**「最大限の対策を講じるが100%の保証はできない」ことをオンボーディングと利用規約に明記**。バッテリー最適化除外・通知許可・自動起動許可の設定をオンボーディングで必須ステップ化する。

## 2. 数学問題エンジン

### 2.1 出題範囲
中学数学／高校数学（数I・数A・数II・数B・数C・数III）を単元単位で選択可能。難易度・出題比率・制限時間・問題数を設定可能（変更なし）。

### 2.2 ★回答形式のルール（v2：統一方針に全面改訂）

★★当初は「整数・有限小数のみ、分数/√/πは出題しない」という制限方針だったが、検討の結果、以下の**統一方針**に全面的に切り替える。

**基本原則**：正解の「真の値」が整数・有限小数・無理数・循環小数のいずれであっても構わない。出題システム内部では正解を厳密な値（シンボリックな構造化データ）として保持する。「きれいな値になるように出題を調整する」という制約は撤廃する。★判定方式の詳細（構造的厳密一致）は本節後半のv3を正とする。

- 回答の入力自体は常に数値（半角数字＋小数点）のみ。分数記号(/)や√記号、π記号そのものを文字として入力させることはしない（記述式にはしない、という当初方針は維持）。
- ★数値入力キーパッドに以下のショートカットボタンを常設し、押すと計算結果の数値がその場で入力欄に挿入される：
  - **π・e**：定数を直接挿入（例：π→3.14159...）
  - **sin・cos・tan**：角度入力（度数法／弧度法切替可）→計算値を挿入（例：tan75°→3.7320508...）
  - **log**：底（10／e(ln)／2／その他）と真数を入力→計算値を挿入（例：log(3)→0.4771212...、ln(3)→1.0986122...）
  - **√**：根号の中身を入力→計算値を挿入（例：√2→1.4142135...、完全平方数でなくてもよい）
  - **分数**：分子/分母を入力→小数に変換して挿入（例：1/3→0.3333...、有限小数か循環小数かを問わず変換する）
- ★★これらのボタンにより、tan75°、log(3)、√2、1/3のような**無理数・循環小数になる値がそのまま最終的な答えになる問題も出題可能**とする。「きれいな値に限定する」という当初の制約(2.2旧版の1.)は撤廃。
- ★★判定方式（v3：構造的厳密一致に修正。epsilon近似判定は撤回）：
  - 出題システムは正解を**厳密な構造化データ**として保持する。例：
    - 整数・有限小数の場合 → `{type: "number", value: 3.25}`
    - 根号の場合 → `{type: "sqrt", arg: 2}`（√2を表す）
    - 三角関数の場合 → `{type: "trig", func: "tan", angle: 75, unit: "deg"}`（tan75°を表す）
    - 対数の場合 → `{type: "log", base: 10, arg: 3}`（log₁₀3を表す）
    - π/e係数の場合 → `{type: "constant_multiple", constant: "pi", coeff: 5}`（5πを表す）
    - 分数の場合 → `{type: "fraction", numerator: 1, denominator: 3}`
  - ★ユーザーの回答入力も、素の数値を手入力する方式（プレーンな数字キー）と、専用ボタン（√・sin/cos/tan・log・π/e・分数）を使って同種の構造を組み立てる方式の2種類がある。
  - ★★**判定は「ユーザーが専用ボタンを使って組み立てた構造」と「正解の構造」を突き合わせて一致するかどうかで行う**（例：正解が`{type:"sqrt", arg:2}`のとき、ユーザーが√ボタンで2を指定していれば正解。一方、√ボタンを使わず素の数字キーで"1.41421356"のように近似値を手入力した場合は、たとえ数値的に近くても**不正解として扱う**）。
  - **数値的な近似一致（epsilon判定）は行わない**。無理数・循環小数を「手入力の近似値」で正解とみなす設計は撤回する。
  - ただし正解自体が整数・有限小数（`{type:"number"}`）の場合は、通常の数値入力と厳密一致（浮動小数点の表現誤差のみを吸収する極小のepsilonは実装上許容）で判定する。これは「無理数を近似で許容する」こととは別問題（浮動小数点の実装上の誤差吸収）である。
  - ★複数の構造が数学的に同値になるケース（例：√8 と 2√2）への対応は本要件では**MVP対象外**とし、出題システム側が常に正規化された唯一の構造（例：根号の中身は必ず最簡形にする）で正解を生成することで回避する。
- ★このボタン方式は、円（中学校数学）、三角比・指数対数（数II）、自然対数・微積分の一部（数III）など、従来「整数・有限小数のみ」という制約と相性が悪かった単元をMVP〜Ver1.1の出題範囲に含めることを可能にする。単元別ジェネレーター側は「きれいな値になるよう逆算する」設計を必須としなくてよくなる（ただし採用してもよい）。
- ★UXトレードオフに関する注記：sin/cos/tan/log/√/分数はボタン操作に角度・真数・分子分母などの追加入力ステップを要するため、π/eの単純挿入よりタップ数が増える。「操作数最小」というUX目標とのバランスを取るため、各ボタンの入力フォームはできる限り最小タップ数（大きな数字キー＋確定ボタンのみ）で設計する。

### 2.3 難易度・時間設計
- 目標解答時間：1問30〜90秒（★ただし数III等、単元特性上この範囲に収まらない場合があることを明記し、単元ごとに現実的な目標値を個別設定する）。

### 2.4 コンポーネント設計（変更なし・第3部で詳細化）

## 4. 音楽連携 ★方針転換

- **アラーム音そのものは常にアプリ内蔵のローカル音源を使用する**（外部ストリーミングサービスの音声を目覚まし用として制御することは技術的・規約的に不可能なため）。
- Spotify / Apple Musicは「起床成功後にご褒美として指定プレイリストを起動する」用途に限定して連携する（Premium会員限定、公式アプリのインストールが前提）。
- YouTube Music / Amazon Musicは公開APIの制約上、本連携の対象外とする。

### 4.1 ★YouTube URL入力によるアラーム音再生（不採用・代替案を採用）
- 検討の結果、**YouTube動画から音声を抽出して自アプリのアラーム音として鳴らす機能は不採用とする**。
  - 理由1（規約）：YouTube利用規約上、動画・音声の抽出/ダウンロードは明確に禁止されている。
  - 理由2（技術）：規約に沿った公式手段（IFrame Player API）は「画面表示中」「オンライン」が必須条件であり、ロック画面上・オフラインでの鳴動というアラームの必須要件を満たせない。
- **代替案として以下を採用する**：
  1. ユーザーが端末内に保存済みの音声ファイル（mp3等）をアラーム音として設定できる機能。
  2. YouTube Musicは「起床成功後にアプリを開いて指定のプレイリスト/動画を再生する」補助連携（Spotify等と同枠組み）に留める。

## 8. プライバシー・法務 ★新規追加
- ターゲットに中学生を含むため、収集する個人情報を最小限にする（学習データ・起床データのみ、氏名等の必須収集は避ける）。
- 広告を導入する場合、**行動ターゲティング広告は子ども向けポリシー上使用しない**（Google Play/Apple両方のポリシー抵触リスク）。
- 利用規約・プライバシーポリシーに、アラームの「起床保証はできない」旨を明記（免責事項）。

---

## 9. 誤答時の挙動 ★新規追加
- 誤答時は再挑戦可能（回数無制限）とし、正答するまで次の問題または停止に進めない。
- ★オプションとして「一定回数誤答した場合に問題を平易なものに自動で差し替える」救済モードを設定可能にする（挫折による二度寝を防ぐため）。
